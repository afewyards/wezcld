# Design: Phase 1.5 — Make split + run real

## Goal

Translate `session split` and `session run` to real `wezterm cli` calls so teammates actually start. Keep logging. All other commands stay fake so we can observe close/focus/shutdown.

## Changes to bin/it2

### session split [-v] [-s <parent>]

```
it2 session split -v           → wezterm cli split-pane --bottom
it2 session split -s <pane-id> → wezterm cli split-pane --pane-id <pane-id> --bottom
```

Returns integer pane ID from wezterm. No mapping layer — Claude Code treats IDs as opaque.

### session run -s <id> '<command>'

```
it2 session run -s <id> '<cmd>' → printf '%s\n' "<cmd>" | wezterm cli send-text --no-paste --pane-id <id>
```

### Everything else

Unchanged — fake responses, keep logging. close/focus/list will now appear in logs during real lifecycle.

## Tests

- Split tests conditional on WezTerm (`$TERM_PROGRAM = WezTerm`)
- Unit tests (version, help, logging) work everywhere
- Live tests (split, run) skipped outside WezTerm
